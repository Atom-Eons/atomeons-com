Persistent inline search bar · full-width under the fixed Header

Operator directive 2026-06-05: "crt k is cool. i want a second full bar
for typing the length of the window under the top nav."

NEW · app/_components/V3/SearchInline.tsx

A "use client" component that renders a sticky, full-width search bar
directly under the fixed Header. Same engine as the ⌘K palette — re-
uses the static /public/search-index.json (86 KB, 196 records) and the
exported rankRecord + fuzzyScore from SearchPalette.tsx.

  - position: sticky · top-16 (slots under the 64px fixed Header)
  - z-30 (Header z-40 stays above, content stays below)
  - bg #08090B/95 + backdrop-blur-md (cinematic ink with proper depth)
  - max-w-[1600px] container — full bleed visual at large viewports
  - mono "::search" eyebrow on desktop, search-icon SVG, native
    <input type="search">, ⌘K / / kbd hint on the right
  - Dropdown results panel absolutely positioned under the bar; shows
    up to 8 hits when query is non-empty + bar is focused; closes on
    click-outside (preserves the query for re-focus)
  - ↑↓ navigate, Enter follows, ESC clears + blurs
  - Lazy-loads the index on first focus OR first non-empty query —
    pages that never engage with search pay zero bytes
  - cache: "force-cache" — second open is 0ms from network
  - Hidden on /admin/* /api/* /auth/* surfaces (no value showing
    search on admin or API routes)

REFACTOR · app/_components/V3/SearchPalette.tsx

Promoted three types and one function to exports so SearchInline can
reuse them without duplicating ~70 lines of fuzzy-matching code:
  + export type SearchRecord
  + export type SearchIndexFile
  + export type Scored
  + export function rankRecord
Internal type aliases (`Record`, `IndexFile`) kept so the rest of the
palette file is unchanged.

WIRING · app/layout.tsx

  + import { SearchInline } from "./_components/V3/SearchInline";
  + Rendered between <Header /> and {children} inside the pt-16
    flex-1 wrapper. Since SearchInline is sticky (not fixed), it
    occupies its own ~52px of in-flow height — no extra padding
    needed.

PERFORMANCE BUDGET

  - Sub-15ms keystroke-to-render. Same numbers as ⌘K (no debounce,
    no Worker, no API roundtrip).
  - First-focus fetch: 30-50ms cold, 0ms cached.
  - Bundle impact: ~3 KB minified (component itself) + 0 KB index
    (already shipped for the palette).
  - Zero CLS — the bar is in flow; no late-injected DOM jumps.

A11Y

  - role="listbox" + role="option" on the dropdown.
  - aria-selected on the active row.
  - aria-label="Search atomeons.com" on the input.
  - <kbd> elements for keyboard hints.
  - Honors prefers-reduced-motion implicitly (no animations to
    disable).
