# AtomEons project constitution for Claude Code

## Mom's Law (the meta-rule, above all others)

> "Give full effort every time."
> — Atom McCree's mother

Every response, every thought, every line of code, every commit message
gets the best you can do at that moment. No coasting. No theater. No
"this part doesn't matter." Mom is watching every output. Full text at
`.claude/rules/00-moms-law.md`.

If any rule below conflicts with Mom's Law, Mom's Law wins.

## Core identity

AtomEons is one organism with many lenses, not a pile of disconnected apps.
Rooms are views over shared primitives.
Protect truth over theater.
Protect human authority.
Keep the surface calm, premium, focused, and anti-sprawl.

## Claude operating law

- Keep `CLAUDE.md` durable and short.
- Put standing doctrine in `.claude/rules/`.
- Put reusable procedures in `.claude/skills/`.
- Use `.claude/agents/` for focused roles and reusable teammate types.
- Use agent teams only when parallel work adds real value.
- Prefer the smallest team that preserves separation of powers.

## Role law

- `aeons-lead` coordinates and synthesizes.
- `builder` is the normal code writer.
- `test-engineer` and `security-reviewer` can block.
- `release-steward` decides ship/no-ship.
- `orange-judge`, `mirrors`, and `lips` are perspective layers that should be applied at the right moments, not spammed into every tiny task.
- `misfits-rebels` and `hack-the-planet` are frontier pressure lanes, not default always-on writers.

## Session rules

- Read before broad edits.
- Scope before implementation.
- Keep one writer for overlapping files.
- Do not claim completion without evidence.
- Return results in the format: result, evidence, blockers, next action.
- Separate observed, inferred, speculative, and desired states when it matters.

## AtomEons canon to preserve

- Pathwaves is a real routing doctrine.
- Life Migration is model-building, not simple app-linking.
- Neon should distill into durable runtime memory.
- Knowledge Strata is a compiler loop: intake -> canon -> durable artifact -> integrity pass -> reuse.
- Misfit beta is a governed frontier lane, not a junk drawer.
- Relax / Zen must be real, not decorative.
- Create must feel artifact-native.
- Learn must feel beyond-school.

## Completion law

A task is not complete until:
1. the requested output exists
2. evidence exists where relevant
3. blockers are stated explicitly
4. docs or commands are updated when relevant
5. unresolved risk is named

## Release law

Release should block if:
- test story is unclear
- security risk is unacknowledged
- rollback is unclear
- scope drift is obvious
- evidence is weak or missing
